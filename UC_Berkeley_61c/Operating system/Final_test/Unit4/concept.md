### 第四章 存储器管理 (全景导读)
**核心议题：** 操作系统如何把有限的物理内存（RAM）分配给多个并发执行的进程，并确保它们互不干扰、高效运行。

#### 1. 存储器的层次结构
这是一个硬件基础背景知识。
*   **物理组成：** 从上到下依次为：**寄存器 (Registers) -> 高速缓存 (Cache) -> 主存 (Main Memory / RAM) -> 磁盘 (Disk)**。
*   **设计原理：** 越靠近 CPU，访问速度越快，但造价越昂贵，容量越小。
*   **操作系统的角色：** 操作系统主要管理的是**主存（内存）**，即负责将磁盘中的程序调入主存，供 CPU（通过 Cache 和寄存器）执行。
*   *考点提要：选择题中常考察这种自顶向下的速度、容量关系。*

#### 2. 程序的装入和链接 (从磁盘到内存的转换)
一个程序（如 C 语言写的代码）编译成机器指令后存放在磁盘上，要运行它，必须将其放入内存。这个过程分为两步：**链接**和**装入**。

**① 链接 (Linking)**
编译器生成的是多个独立的目标模块（`.o` 文件），链接的作用是将它们拼接成一个完整的可执行文件，并解析模块间的相对地址。
*   **静态链接：** 在程序运行之前，就把所有模块和库函数完整地拼接在一起。缺点是占用磁盘和内存空间大。
*   **装入时动态链接：** 在将模块装入内存时，边装入边链接。
*   **运行时动态链接：** 在程序执行期间，当执行到需要某个模块的代码时，才去链接该模块。优点：节省内存，便于更新库文件。

**② 装入 (Loading) - 【重点：地址转换机制】**
编译器生成的指令中，使用的地址是从 `0` 开始的**逻辑地址（相对地址）**。但主存中的实际存放位置是**物理地址（绝对地址）**。装入的过程，核心就是**地址转换**。
*   **绝对装入：** 在编译时就确定程序将放在内存的哪个物理地址。装入时直接原封不动放进去。仅适用于单道程序环境（非常古老）。
*   **可重定位装入（静态重定位）：** 在程序装入内存的**那一刻**，由装入程序将代码中所有的逻辑地址一次性修改为物理地址。
    *   *缺点：* 一旦装入，程序在内存中绝对不能移动位置，也不能在运行时申请新的内存空间。
*   **动态运行时装入（动态重定位）：** **【必考】** 程序装入内存后，指令中依然保留逻辑地址。真正的地址转换推迟到 CPU **执行每一条指令的时刻**才进行。
    *   *硬件支持：* 必须依赖**重定位寄存器（也叫基址寄存器）**。
    *   *原理：* `物理地址 = 逻辑地址 + 重定位寄存器中的值`。
    *   *优点：* 允许程序在内存中移动位置，是现代操作系统的基础。

#### 3. 连续分配存储管理方式
这是早期操作系统的内存分配方式：**一个进程必须占用一段连续的物理内存空间**。

*   **单一连续分配 / 固定分区分配：** 提前把内存划分好区域，一个进程放一个区域。
    *   *核心缺陷 - **内部碎片**：* 分配给进程的内存区域大于进程实际需要的空间，这部分多出来的空间在区域内部，无法被其他进程使用。

*   **动态分区分配：** **【计算题/算法题考点】**
    系统不预先划分内存，而是当进程进入时，根据进程的确切大小，动态地从空闲内存中切出一块连续的空间分配给它。
    *   *核心缺陷 - **外部碎片**：* 随着进程的分配和释放，内存中会产生许多太小而无法满足任何新进程需求的空闲小区域。解决外部碎片的方法是**紧凑（Compaction，即把进程移动到一端，合并空闲区，这需要动态重定位的支持）**。
    *   **四种分配算法（决定把哪个空闲分区分配给进程）：**
        1.  **首次适应 (First Fit, FF)：** 将空闲分区按**起始物理地址递增**的顺序链接。每次从头开始查找，找到第一个能满足要求的空闲分区就分配。*（工程上最高效）*
        2.  **邻近适应 (Next Fit, NF)：** 从上一次查找结束的位置开始继续查找。
        3.  **最佳适应 (Best Fit, BF)：** 将空闲分区按**容量大小递增**的顺序链接。找到的第一个满足要求的必定是最小的可用分区。*（缺点：会产生极其微小的外部碎片）*
        4.  **最坏适应 (Worst Fit, WF)：** 将空闲分区按**容量大小递减**的顺序链接。每次直接取最大的空闲分区切分。*（缺点：快速消耗掉大块连续内存）*

#### 4. 内存管理的基本概念
对上述内容的理论抽象，常作为简答题或名词解释。
*   **逻辑地址空间与物理地址空间：** 逻辑地址是 CPU 生成的地址，物理地址是内存硬件实际的地址。
*   **内存保护：** 确保一个进程不能访问（读/写）属于操作系统的或其他进程的内存空间。
    *   *实现方式：* 通常通过一对硬件寄存器：**基址寄存器**（保存进程起始物理地址）和**限长寄存器**（保存进程的最大逻辑地址长度）。CPU 每次访存都会检查：`逻辑地址 < 限长寄存器`，若通过，再计算物理地址。

> **下面进入第四章的深水区（5和6）。为了解决“连续分配”带来的严重碎片问题，现代操作系统引入了“离散分配”机制，即：分页与分段。**

#### 5. ★★ 分页式存储管理 (Paging)
**核心原理：**
将物理内存划分为大小相等的、固定大小的块，称为**物理块 (Frame，或页框)**。
将进程的逻辑地址空间也划分为与物理块大小完全相同的块，称为**页 (Page)**。
分配内存时，一页可以被装入任何一个不相邻的物理块中。（解决了外部碎片问题，但最后一页可能装不满，存在少量内部碎片）。

**① 页表结构与地址转换 (Address Translation)**
为了知道程序的哪一“页”存放在内存的哪一“块”，操作系统为每个进程维护一个数据结构：**页表 (Page Table)**。
*   **页表的本质：** 一个数组。索引（行号）是页号 $P$，数组中存储的值是对应的物理块号 $F$。

**【重点：分页地址转换计算步骤】**
假设给定一个逻辑地址，系统如何计算出物理地址？
1.  **拆分逻辑地址：** 逻辑地址由两部分组成：**[页号 P] + [页内偏移量 W]**。
    *   如果页面大小是 $2^n$ 字节，那么逻辑地址的**低 $n$ 位就是偏移量 $W$**，**高位部分就是页号 $P$**。
    *   *数学计算：* $P = \text{逻辑地址} \div \text{页面大小}$（取整数商）； $W = \text{逻辑地址} \bmod \text{页面大小}$（取余数）。
2.  **越界检查：** 判断计算出的页号 $P$ 是否大于等于 **页表长度**（系统寄存器中保存）。如果是，触发越界中断。
3.  **查询页表：** 根据页号 $P$，在页表中找到对应的表项，读出**物理块号 $F$**。
4.  **计算物理地址：**
    *   `物理地址 = 物理块号 F * 页面大小 + 页内偏移量 W`。
    *   *(在二进制下，就是直接把物理块号 $F$ 作为高位，拼接上低位的偏移量 $W$)*。

**② TLB 快表 (Translation Lookaside Buffer)**
*   **为什么需要 TLB？** 根据上面的步骤，每次访问数据，CPU 实际上要访问**两次**主存：第一次访问内存中的页表获取块号，第二次访问该块号对应的内存读取数据。这太慢了。
*   **TLB 的作用：** TLB 位于 CPU 内部，是一种极高速的硬件缓存，专门用来存放**最近访问过的页表项**。
*   **计算：有效访问时间 (Effective Access Time, EAT)**
    *   设 TLB 查找时间为 $\lambda$（极短），内存访问时间为 $t$（较长），TLB 命中率为 $a$。
    *   **若 TLB 命中 (概率 $a$)：** 去 TLB 查到块号，直接去内存读数据。总时间 = $\lambda + t$。
    *   **若 TLB 未命中 (概率 $1-a$)：** 去 TLB 没查到，只能去内存查页表，查到后再去内存读数据。总时间 = $\lambda + t + t = \lambda + 2t$。
    *   **EAT 公式 = $a \times (\lambda + t) + (1 - a) \times (\lambda + 2t)$**。

#### 6. ★ 分段式存储管理 (Segmentation)
**核心原理：**
分页是从“计算机管理内存”的角度设计的（固定大小）。而分段是从“程序员”的角度设计的。
程序是由多个逻辑模块组成的（如：主程序代码段、数据段、栈段）。分段机制就是允许将这些逻辑段分别独立地分配到内存的不同区域。
*   *区别：* 分页的页大小是固定的，而**分段的段大小是可变的**（代码段可能很大，数据段可能很小）。

**① 段表与地址转换**
因为每个段的长度不一样，所以**段表 (Segment Table)** 中，每个表项必须包含两个信息：该段在内存中的**基址 (Base)**，以及该段的**段长 (Limit)**。

**【重点：分段地址转换计算步骤】**
1.  **拆分逻辑地址：** 逻辑地址由 **[段号 S] + [段内偏移量 W]** 组成。
2.  **段号越界检查：** 判断段号 $S$ 是否大于等于 段表长度。若是，触发中断。
3.  **查询段表：** 根据段号 $S$，找到段表项，提取出该段的 **段长 Limit** 和 **基址 Base**。
4.  **【核心差异：段内偏移越界检查】：** 必须比较偏移量 $W$ 和段长 Limit。**如果 $W \ge Limit$，说明试图访问超出该段长度的内存，触发越界中断**。*(注：分页机制中没有这一步，因为页大小固定，偏移量不可能越界)*。
5.  **计算物理地址：** `物理地址 = 基址 Base + 段内偏移量 W`。

**② 分页 vs 分段的严谨比较 (经典论述题重点)**
1.  **信息单位：** 页是物理单位（固定大小，系统需求）；段是逻辑单位（可变大小，用户需求）。
2.  **地址空间维度：** 分页是一维的（只需给出一个线性地址，硬件自动拆分）；分段是二维的（程序员或编译器必须明确指定“段号”和“段内偏移”）。
3.  **碎片问题：** 分页没有外部碎片，但会产生**内部碎片**（最后一页装不满）；分段没有内部碎片（按需切分），但会产生**外部碎片**（段与段之间的空隙）。
4.  **共享与保护：** 分段比分页更容易实现逻辑模块的共享和保护（例如，可以把某个代码段标记为只读并共享给多个进程，分页很难做到这么精确的逻辑划分）。
