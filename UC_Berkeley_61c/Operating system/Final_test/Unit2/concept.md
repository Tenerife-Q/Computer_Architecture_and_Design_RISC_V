## 知识点 1：前趋图和程序执行（考点深度拆解）

### 1. 顺序执行的特征：为什么“可再现”？

* **封闭性**：程序在运行时，**独占**系统中的所有资源。除了它自己，没有其他程序能改变这些资源的状态。
* **可再现性**：只要输入的初始数据相同，无论你今天运行、明天运行，还是中间暂停一下再运行，结果**必然一模一样**。

### 2. 并发执行的特征：为什么“失去封闭性与可再现性”？

* **间断性（走走停停）**：因为共享资源和 CPU 切换，程序执行变成了“执行一下 -> 停下来等资源 -> 再执行”。
* **失去封闭性**：内存里的变量不仅你在改，别人也在改。
* **失去可再现性**：由于执行顺序受到了并发调度的随机影响，**相同的输入可能导致截然不同的输出**。这也是为什么要引入“进程同步”的根本原因。

## 知识点 2：★ 进程的描述（高频核心考点）

### 1. 深入 PCB（进程控制块）

大纲里提到 PCB，选择题最喜欢考“PCB 里到底有什么？”和“它怎么组织？”：

* **内部信息**：
1. **进程标识符（PID）**：类似于身份证号。
2. **处理机状态（寄存器上下文）**：CPU 切换时，必须把当前寄存器的值、程序计数器（PC）存在 PCB 里，下次才能接着读。
3. **进程调度信息**：进程状态、优先级。
4. **资源与控制信息**：内存地址空间指针、打开的文件列表。


* **组织方式**：
* **链接方式**：把相同状态的 PCB 链接成一个队列（如就绪队列、阻塞队列）。
* **索引方式**：建立索引表，记录各状态 PCB 的起始地址。



### 2. 状态转换的底层魔鬼细节（必考！）

结合上面的三状态转换，考试会挖以下细节坑：

* **就绪 $\rightarrow$ 执行**：进程调度程序选中了该进程，分配了 CPU 时间片。
* **执行 $\rightarrow$ 就绪**：
1. **时间片用完**。
2. **被高优先级进程抢占**。


* **执行 $\rightarrow$ 阻塞**：进程**主动**调用了阻塞原语（如等待 I/O 读写、等待信号量）。**注意：进程无法自己把自己唤醒！**
* **阻塞 $\rightarrow$ 就绪**：I/O 操作完成或等待的事件发生。由**外部中断/其他进程**触发唤醒原语。

## 知识点 3：★ 进程控制（原语的真相）

### 1. 为什么原语必须用“关中断/开中断”实现？

原语（Primitive）是管态（内核态）下执行的特权指令序列。

* **原理**：当 CPU 执行**关中断**指令后，它就会无视任何外部硬件发出的中断信号（比如时钟中断、键盘输入）。这样，CPU 就会一口气把接下来的进程控制代码执行完，最后执行**开中断**恢复正常。
* 由于中途无法发生时钟中断，CPU 就绝对不会发生进程切换，从而保证了“原子性（不可分割）”。

### 2. 进程创建（Creation）的步骤

考试如果考简答题，问“创建一个进程需要几步？”，标准答案：

1. **申请空白 PCB**（若申请失败，说明系统进程数到上限了）。
2. **为进程分配资源**（内存、物理空间等）。
3. **初始化 PCB**（设置 PID、初始化寄存器和状态等）。
4. **将 PCB 插入就绪队列**。

## 知识点 4：线程概念（终极辨析题对决）

选择题极爱出“下列关于进程和线程的叙述中，正确的是...”这类题。请务必记住这四张“对比王牌”：

| 属性 | 进程（Process） | 线程（Thread） |
| --- | --- | --- |
| **根本定义** | 系统**资源分配**的基本单位 | **资源调度和执行**的基本单位 |
| **并发性** | 进程之间可以并发 | 线程之间、进程之间都能并发（并发度更高） |
| **拥有资源** | 拥有独立的内存、文件等资源 | 自己不拥有资源，共享所属进程的全部资源 |
| **系统开销** | **大**。切换时要销毁/创建整个地址空间，刷新 TLB/缓存 | **小**。同一进程内的线程切换，不需要切换地址空间 |

> **关键易错点**：同一个进程内的所有线程，它们**共享**该进程的：
> * 代码段、全局变量、堆（Heap）、打开的文件。
> 
> 
> 但它们**独享**（不共享）的有：
> * **线程栈（Stack）**：局部变量和函数调用轨迹不能混。
> * **程序计数器（PC）和寄存器**：各自执行到哪一步必须自己记着。
> 
>

## 知识点 5：★★ 进程同步（前半部分理论攻克）

大题留在下一阶段，我们先通过计算和概念把“整型”和“记录型”信号量吃透。

### 1. 整型信号量 vs 记录型信号量

* **整型信号量 $S$**：就是个整数。
* `wait(S)` 操作（即 P 操作）：`while(S <= 0); S--;`
* **致命缺陷**：如果 $S \le 0$，进程就会在 `while` 那里死循环。这就叫“忙等”（Busy waiting），严重违反了“让权等待”原则，白白浪费 CPU。


* **记录型信号量**：除了一个整型变量 `value`，还带了一个**链表指针 `L**`（用于链接所有等待该资源的进程）。
```c
typedef struct {
    int value;
    struct process_control_block *list;
} semaphore;

```



### 2. 记录型信号量 `wait` 和 `signal` 的严密数学含义（必考选择/计算）

#### `wait(S)`（即 P 操作，申请资源）：

```c
void wait(semaphore S) {
    S.value--;
    if (S.value < 0) {
        block(S.list); // 自己去排队睡觉
    }
}

```

* **考点**：`S.value` **先减 1**。如果减完之后发现 $< 0$，说明前面已经没资源了，当即调用 `block` 原语强行让自己进入阻塞队列。

#### `signal(S)`（即 V 操作，释放资源）：

```c
void signal(semaphore S) {
    S.value++;
    if (S.value <= 0) {
        wakeup(S.list); // 从排队队列里叫醒一个
    }
}

```

* **考点**：`S.value` **先加 1**。加完之后，如果发现结果仍然 $\le 0$，这意味着什么？意味着**原本的值是负数，外面正有兄弟在排队卡着呢**！所以必须调用 `wakeup` 原语把队列里的第一个进程叫醒，送去就绪队列。

### 一、 程序的执行与进程的概念（常考：选择、判断、简答）

**1. 程序的顺序执行 vs 并发执行**
*   **【判断/选择】特征对比**：
    *   顺序执行：具有**封闭性**（独占资源）、**可再现性**（结果确定）。
    *   并发执行：**失去封闭性**、**失去可再现性**、具有**间断性**（走走停停）。
*   **【简答/计算】Bernstein 条件**：判断两个程序能否并发的依据。口诀：“**读写交集必须为空**”。只要一个程序写了某个变量，另一个程序读或写了同一个变量，就绝对不能并发。

**2. 进程与程序的区别（经典简答题）**
*   **【简答/判断】四大区别**：
    1.  **动态 vs 静态**：进程是动态的（程序的执行过程），程序是静态的（磁盘上的代码）。
    2.  **生命周期**：进程有创建、运行和消亡；程序是永久存在的。
    3.  **组成**：进程 = 程序 + 数据 + **PCB**。
    4.  **对应关系**：一个程序可以产生多个进程（比如多开微信），一个进程也可以执行多个程序。

**3. PCB（进程控制块）**
*   **【填空/选择】核心地位**：PCB 是**进程存在的唯一标志**！创建进程就是创建 PCB，撤销进程就是撤销 PCB。
*   **【选择】PCB 包含什么**：PID（进程标识符）、进程状态、处理机状态（各种寄存器上下文，切换时要保存）、调度信息。

---

### 二、 进程的状态与转换（极高频考区：选择、填空、画图题）

这是客观题的“重灾区”，必须形成条件反射！

**1. 三大基本状态流转（箭头方向）**
*   **【选择/判断】必然考点**：
    *   **运行 $\rightarrow$ 就绪**：**时间片用完**，或被高优先级进程抢占。（被动）
    *   **运行 $\rightarrow$ 阻塞**：请求 I/O 操作，或申请临界资源失败。（**主动**让权）
    *   **阻塞 $\rightarrow$ 就绪**：I/O 操作完成，或等到了资源。（被动唤醒）
    *   ⚠️ **绝对雷区**：**阻塞绝不能直接到运行！** 必须先进入就绪队列排队。

**2. 挂起状态（Suspend）**
*   **【简答/选择】为什么挂起？**：内存不足，操作系统把暂时不运行的进程数据**踢到外存（磁盘）**上，腾出内存空间。
*   **【简答】阻塞 vs 挂起**：
    *   **阻塞**：进程还在**内存**中，只是在等资源。
    *   **挂起**：进程被放到了**外存（磁盘）**中，处于静止状态。

---

### 三、 进程控制与原语（常考：选择、填空）

**1. 原语（Primitive）**
*   **【填空/判断】原子性**：原语是操作系统内核中执行特定功能的一段程序，**执行过程不可中断**（一气呵成）。
*   **【填空】底层实现**：原语的不可中断性是通过硬件的 **“关中断” 和 “开中断”** 指令实现的。
*   **【选择】运行环境**：原语特权极高，只能在操作系统的 **内核态（核心态）** 运行，不能在用户态运行。

**2. 进程切换**
*   **【简答】进程切换到底干了啥？**：
    1. 保存当前进程的 CPU 上下文（PC、寄存器）到它的 PCB 中。
    2. 从就绪队列选一个新进程。
    3. 把新进程 PCB 里的上下文恢复到 CPU 寄存器里。

---

### 四、 线程的概念（必考：选择、判断、简答）

这是现代操作系统最重要的概念对比。

**1. 进程 vs 线程（经典简答题）**
*   **【核心填空/判断】本质区别**：
    *   **进程** 是操作系统**分配资源**的基本单位。
    *   **线程** 是 CPU **调度和分派**的基本单位。
*   **【选择】开销对比**：线程自己只有很小的 TCB（线程控制块）和独立的**栈（Stack）**。同进程内的线程切换，不需要切换内存页表等虚拟地址空间，所以**开销极小**。

**2. 用户级线程 (ULT) vs 内核级线程 (KLT)**
*   **【选择/简答】致命考点**：
    *   **用户级线程（ULT）**：由应用程序自己管，内核不知道。**缺点**：如果一个 ULT 发起系统调用被阻塞，内核会以为整个进程阻塞了，导致**该进程下所有线程全部卡死**。优点：切换极快（不用进内核态）。
    *   **内核级线程（KLT）**：由操作系统内核管。**优点**：一个线程阻塞，其他线程不受影响；能利用多核 CPU 真正并行。缺点：切换慢（需要进入内核态）。

### 五、 进程同步基础概念（在大题外，常考选择/填空）

在让你写伪代码大题之前，老师一定会考考你底层的概念准则。

**1. 临界区四大准则**
*   **【选择/判断/简答】必须背熟**：
    1.  **空闲让进**：无人在临界区，允许进入。
    2.  **忙则等待**：有人在临界区，必须等待。
    3.  **有限等待**：保证有限时间内能进入，防止饥饿。
    4.  **让权等待**：进不去时，应**立即释放 CPU**（阻塞自己），**严禁“忙等”**。

**2. 信号量机制的底层区别**
*   **【选择/判断】整型 vs 记录型信号量**：
    *   **整型信号量**：用 `while(S <= 0);` 循环等待。**致命缺陷：违反了“让权等待”准则**，会造成 CPU 盲等浪费。
    *   **记录型信号量**：自带一个阻塞队列。进不去就调用 `block()` 阻塞自己，**遵循了“让权等待”**。

**3. 信号量 $S$ 的物理意义**
*   **【填空/选择】绝对高频考点**：
    *   $S > 0$：表示系统中还有多少个可用资源。
    *   $S < 0$：表示资源已耗尽，且其绝对值 $|S|$ 表示**当前阻塞队列中有多少个进程正在排队等待**。
    *   *(例如：$S = -3$，填空题答案填：当前有 3 个进程处于阻塞状态)*。
