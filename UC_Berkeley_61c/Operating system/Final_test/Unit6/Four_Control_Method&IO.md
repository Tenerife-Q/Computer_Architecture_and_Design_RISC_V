# 第1关：第六章 设备管理（上）—— I/O系统与控制方式

**【核心目标】** 搞懂CPU（大老板）是怎么管理手下成百上千个慢吞吞的外部设备（员工）的。

### 1. I/O 系统基础 (大纲要求：设备分类、设备控制器)

**① 设备分类（常考选择题）**
你只需要记住最核心的一种分类方式：按**数据传输单位**分类。
*   **块设备 (Block Device)：** 数据是一块一块传输的。有地址，可以随机访问任何一个块。
    *   *例子：* **磁盘**（硬盘）、U盘。 （记住：磁盘是块设备的老大）。
    *   *特点：* 传输速度快，可寻址。
*   **字符设备 (Character Device)：** 数据是一个字符一个字符传输的。没有地址，只能顺序读写。
    *   *例子：* 键盘、鼠标、打印机。
    *   *特点：* 传输速度慢，不可寻址。

**② 设备控制器（了解概念）**
*   **大白话解释：** CPU是总经理，他太忙了，不可能亲自去教打印机怎么喷墨，教硬盘怎么转圈。所以CPU需要一个“部门经理”来传达命令，这个部门经理就是**设备控制器**。
*   **作用：** 接收CPU的指令，将其翻译成设备能听懂的电信号，控制设备工作。它是CPU和机械设备之间的桥梁。

### 2. I/O 控制方式 (大纲要求：程序查询、中断I/O、DMA、通道) —— 【必考客观题/简答题】

这是本节的**绝对重点**！考试特别喜欢考这四种方式的**进化过程**。它们进化的唯一动力就是：**尽量解放CPU，让CPU少管闲事（少参与I/O），多去算数。**

**① 程序查询方式 (轮询)**
*   **工作原理：** 就像你用锅烧水，你什么都不干，就死死盯着锅，一直问“水开了吗？水开了吗？”直到水开。
*   **专业解释：** CPU不断地去读取设备的状态寄存器，看看设备准备好数据没有。
*   **缺点：** CPU**严重浪费**，一直处于“盲等”状态，效率极低。

**② 中断驱动 I/O (Interrupt)**
*   **工作原理：** 就像你用带响铃的水壶烧水。你把水烧上，就去打游戏（CPU运行其他进程）。水烧开了，水壶发出“呜呜”声（发出中断信号），你听到后暂停游戏，去把火关了。
*   **专业解释：** 设备准备好数据后，主动向CPU发送中断信号。CPU收到后停下手头工作，把数据**一个字一个字**地从控制器读到内存中。
*   **缺点：** 每次只能传输很小的数据（比如一个字）。如果大量传数据（比如拷电影），设备会频繁打断CPU，CPU还是很累。

**③ DMA方式 (Direct Memory Access，直接存储器存取) —— 【高频考点】**
*   **工作原理：** 就像你雇了一个钟点工（DMA控制器）。你告诉她：“把这桶水倒进缸里，倒完了叫我”。钟点工自己一瓢一瓢倒（数据直接在硬盘和内存之间传输），全程不需要你动手。倒完后，她喊你一声（发一次中断）。
*   **专业解释：** 数据**直接在内存和外部设备之间传输**，不需要经过CPU。CPU只在数据块传输的**开始和结束**时才干预。
*   **优点：** 传输单位变成了**“数据块”**，大大减少了中断CPU的次数。
*   **常考题眼：** DMA方式传输数据不经过CPU，直接在内存和设备间进行；DMA只在数据块传输的始末才需要CPU干预。

**④ 通道控制方式 (Channel)**
*   **工作原理：** 就像你直接请了一个“全职大管家”（通道）。你扔给管家一份清单：“今天去买菜、洗衣服、做饭”，然后你就不管了。管家自己安排所有的事，全部搞定了再向你汇报。
*   **专业解释：** 通道是一个**弱化版的CPU**（专门负责I/O的处理器）。它可以执行自己的通道程序，独立完成一系列复杂的I/O操作。
*   **对比DMA：** DMA只能控制一台或几台同类设备，传一个数据块；通道可以控制多台不同设备，执行一系列任务。
