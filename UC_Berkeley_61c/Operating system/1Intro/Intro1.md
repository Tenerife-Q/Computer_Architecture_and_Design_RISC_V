### 第一阶段：机器唤醒的底层逻辑 (What Happens at Boot? - 图1、图4)

在通电的瞬间（T=0），CPU 的所有通用寄存器清零，内存（DRAM）中全是随机乱码，没有任何有效的操作系统。CPU 必须要有第一条指令才能开始运转。

#### 1. 硬件级寻址与只读存储器（ROM）
*   **数字逻辑状态**：当主板发出 Reset（重置）信号，CPU 内部的控制电路会将**程序计数器（Program Counter, PC）**强制赋予一个硬件写死的固定物理地址（幻灯片中举例为 `PC = 0x2000`）。
*   **内存映射（Memory Mapping）**：请看图4右侧的图解。地址 `0x2000` 实际上**并不指向**主板上的内存条（DRAM），而是通过系统的地址总线，被映射到了主板上一块独立的非易失性芯片（Flash ROM/BIOS芯片）。
*   **执行第一条指令**：CPU 取出 `0x2000` 地址处的指令并执行。这就是机器的第一行代码。

#### 2. 开机控制权的转移链 (Boot Sequence 逻辑流)
整个开机过程是一个“**低级代码加载高级代码，然后交出控制权**”的接力过程：

```text
[控制权接力流]
1. CPU 硬件 (PC 强制指向 ROM)
   ↓
2. BIOS/UEFI (运行在 ROM 中)
   |-- 扫描硬件总线（PCIe, USB等）
   |-- 定位外部存储设备（如 磁盘盘符 0 的扇区 0）
   ↓
3. Bootloader (运行在 DRAM 中)
   |-- BIOS 将磁盘第一个扇区（通常是 512 字节的 Bootloader，如 GRUB）复制到 DRAM 中。
   |-- BIOS 将 PC 指针跳转到该 DRAM 地址。
   |-- Bootloader 的核心任务：识别文件系统，找到庞大的 OS 内核文件（Kernel）。
   ↓
4. 操作系统 Kernel (运行在 DRAM 中)
   |-- Bootloader 将内核读取并加载到 DRAM 的高地址区。
   |-- Bootloader 将 PC 指针跳转到内核的入口点（Entry Point）。
   |-- OS 接管系统，初始化内存管理（MMU）、中断向量表，最后启动第一个用户级进程（Init/终端）。
```

### 第二阶段：启动应用程序的内存架构 (Launching Applications - 图5)

当操作系统运行起来后，你通过命令行（Shell）输入一个命令（比如 `./my_program`），OS 是如何把硬盘里的静态文件变成一个活生生在跑的“进程（Process）”的？

#### 1. 进程 (Process) 架构
在现代 OS 中，**进程是资源分配的基本单位**。
*   **独立性**：每个进程拥有自己独立的“虚拟地址空间”（后续课程的核心）。进程A 绝对无法用普通的 Load/Store 指令读取进程B 的内存。
*   **线程 (Thread)**：线程是 CPU 调度的基本单位。一个进程内可以有多个线程，它们**共享**同一个地址空间（能互相读写变量）。

#### 2. Linux 下的启动逻辑 (fork & execve)
图5提到了 Linux 中特有的启动逻辑，这实际上分为两步独立的系统调用：
*   **`fork()`**：当前的 Shell 进程克隆一个和自己一模一样的子进程。
*   **`execve()`**：子进程调用此指令，将自己内存里的内容“清空”，并用硬盘上 `./my_program` 的代码和数据来**覆写**自己。

#### 3. 应用程序的内存布局初始化 (Memory Layout)
当 `execve` 执行时，OS 会从硬盘读取可执行文件，并在 DRAM 中为这个新进程建立严格的内存布局结构（从低地址到高地址）：

```text
================ 高地址 (例如 0x7FFF FFFF) ================
|  栈 (Stack)      | <- 存放局部变量、函数调用的返回地址。向下生长 (地址减小)
|  ↓ ↓ ↓           |
|------------------|
|  (未分配的自由空间)|
|------------------|
|  ↑ ↑ ↑           |
|  堆 (Heap)       | <- 存放 malloc() 动态分配的内存。向上生长 (地址增加)
|------------------|
|  未初始化数据 (.bss)| <- 全局变量 (未赋初值的)
|------------------|
|  已初始化数据 (.data)| <- 全局变量 (如 int x = 5;)
|------------------|
|  代码段 (.text)    | <- 存放机器指令。只读！防止程序篡改自己的代码
================ 低地址 (例如 0x0040 0000) ================
```

#### 4. 移交 CPU 给应用程序
当上述内存布局搭建完毕后，OS 会做最后两件事：
1.  **设置参数**：将命令行的参数（`argc`, `argv`）压入刚刚建好的**栈（Stack）**中。
2.  **修改 PC 指针**：OS 将 CPU 的 PC 寄存器强制设定为上述内存布局中 **代码段（.text）** 的起始地址，即 `main()` 函数的第一条指令所在的地址。

至此，应用程序正式开始在 CPU 上一个时钟周期接一个时钟周期地执行。
