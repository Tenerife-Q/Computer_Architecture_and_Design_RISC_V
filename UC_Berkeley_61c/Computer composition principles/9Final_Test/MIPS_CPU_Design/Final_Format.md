### 🧠 深度解析与标准答案（请做完后再看对照！）

这道题你不用死记硬背，只要在心里默念这几个问题，答案自己就出来了：

#### 1. 灵魂第一问：要不要写寄存器？(`RegWrite`)
*   **不写的只有三类**：往内存存数据的 `sw`、只管跳转的 `beq` 和 `j`。这三个填 `0`。
*   剩下的 `add`, `addi`, `lw` 全都要把结果写回寄存器，填 `1`。
*   **💥 核心连招法则（极其重要）：只要 `RegWrite = 0`，那么“写哪个寄存器(`RegDst`)”和“把什么写进去(`MemtoReg`)”就失去了意义，这俩必须填 `X`！**

#### 2. 灵魂第二问：要不要碰内存？(`MemRead` / `MemWrite`)
*   **读内存 (`MemRead`)**：只有 `lw` 填 `1`，其他全填 `0`。
*   **写内存 (`MemWrite`)**：只有 `sw` 填 `1`，其他全填 `0`。

#### 3. 灵魂第三问：ALU 的第二个输入选谁？(`ALUSrc`)
*(提示：0 选寄存器 rt 的值，1 选指令里带的立即数 imm)*
*   **选寄存器的 (填 0)**：`add` (算 rs+rt)；**`beq` (算 rs-rt，专门挖坑的地方！)**。
*   **选立即数的 (填 1)**：`lw` / `sw` (算 rs+imm 地址)；`addi` (算 rs+imm)。
*   **特殊情况**：`j` 指令根本不用主 ALU，所以它的 `ALUSrc` 填 **`X`**！

#### 4. 灵魂第四问：如果要写寄存器，写给谁？写什么？(`RegDst` / `MemtoReg`)
*   **`RegDst` (0写给rt，1写给rd)**：`add` 有 rd 所以填 `1`；`lw` 和 `addi` 只有 rt 所以填 `0`。不写的填 `X`。
*   **`MemtoReg` (0写ALU的结果，1写内存读出的结果)**：只有 `lw` 是来自内存填 `1`；`add` 和 `addi` 都是 ALU 算出来的填 `0`。不写的填 `X`。

### 👑 压轴大题模拟：第二问（数据通路追踪与计算）

**【题目描述】**
某 32 位 MIPS 单周期处理器，其主存储器按**字节**编址。已知在某时刻，相关通用寄存器内的十进制数值为：**$rs = 20$，$rt = 20$，$rd = 15$**。
指令中提取出的立即数对应的十进制数值为：**$offset = 12$，$label = -3$**。

当前存储器的部分片段如下表所示。请分别分析下列 3 种**独立执行**的条件下，指令执行结束后相关寄存器的值。（注：结果可以用十六进制或十进制表示，请注明清楚）

| 存储单元地址 (十六进制) | 存储单元内容 (指令或数据) |
| :--- | :--- |
| `0x00400010` | `add rd, rs, rt` |
| `0x00400014` | `lw rt, offset(rs)` |
| `0x00400018` | `beq rs, rt, label` |
| ...... | ...... |
| `0x00000018` | `0x000000AA` |
| `0x00000020` | `0x000000FF` |
| `0x00000024` | `0x000000BB` |

**请回答下列问题：**
（1）若执行 `add` 指令，执行结束后，**PC** 和 **rd** 寄存器中的内容分别是什么？
（2）若执行 `lw` 指令，执行结束后，**PC** 和 **rt** 寄存器中的内容分别是什么？
（3）若执行 `beq` 指令，执行结束后，**PC** 寄存器的内容是什么？

---

### 🧠 深度解析与标准答案（一定要自己先算一下再看！）

做这种题，核心口诀是：**“PC加4是本能，十进制十六进制别搞混！”**

#### （1）解析 `add` 指令
*   **算 PC：** 顺序执行，当前 PC 是 `0x00400010`，加上 4 个字节，结果为 **`0x00400014`**。
*   **算 rd：** 把 rs 和 rt 的值相加。$20 + 20 = 40$（十进制）。如果转成十六进制就是 `0x28`。
*   **✅ 答案 (1)：PC = `0x00400014`，rd = `40` (或 `0x00000028`)。**

#### （2）解析 `lw` 指令 🚨【连环陷阱预警】🚨
*   **算 PC：** 只要不是跳转成功，PC 永远加 4。当前 PC 是 `0x00400014`，加 4 变成 **`0x00400018`**。
*   **算 rt（注意这里的十/十六进制转换）：**
    1.  算物理地址：$rs + offset = 20 + 12 = 32$（**注意：这是十进制的 32！**）
    2.  查表陷阱：千万不要去表里找 `0x00000032`！必须把十进制的 32 转换成十六进制，$32 \div 16 = 2$，所以对应的十六进制地址是 **`0x00000020`**。
    3.  查表：地址 `0x00000020` 里的内容是 `0x000000FF`。这才是真正装进 rt 的数据。
*   **✅ 答案 (2)：PC = `0x00400018`，rt = `0x000000FF` (或十进制 `255`)。**

#### （3）解析 `beq` 指令 🚨【负数偏移量陷阱】🚨
*   **判断条件：** 当前 $rs = 20, rt = 20$。两者相等，**条件成立，发生跳转！**
*   **算 PC：** 
    1.  先算基准值：当前 `PC + 4 = 0x00400018 + 4 = 0x0040001C`。
    2.  算偏移量：题目给的 $label = -3$。这意味着要**往回跳** 3 条指令。偏移字节数 = $-3 \times 4 = -12$（十进制）。
    3.  最终 PC = `0x0040001C` 减去十进制的 12（即十六进制的 `0xC`）。
    4.  计算：`1C - C = 10`。所以最终地址为 **`0x00400010`**。（相当于这其实是个 `while` 循环，跳回了第一条 `add` 指令）。
*   **✅ 答案 (3)：PC = `0x00400010`。**

### 💡 考场反杀总结

如果考试中出现这个“第二小问”，阅卷老师就在等那些**粗心大意**的人掉进这三个坑：
1. **查表不转进制**：算出来地址是 32，直接去找表格里的 32，白白丢分（必须转成 `0x20`）。
2. **PC 忘了先加 4**：算跳转目标时，直接拿当前 PC 加上偏移量，错！（必须是 `PC+4` 再加偏移量）。
3. **偏移量忘了乘 4**：算跳转地址时，直接加上了 -3，错！（指令字长 4 字节，必须乘以 4 变成 -12）。

### 👑 压轴大题模拟：第三问（写出指令周期与微操作流程）

**【答题规范提醒（阅卷老师最看重这三点）】：**
1. **周期名称必须标出**：取指(IF)、译码(ID)、执行(EX)、访存(MEM)、写回(WB)。
2. **区分寄存器和内存**：通用寄存器通常写成 `R[rs]` 或 `(rs)`；内存必须写成 `M(MAR)` 或 `Mem[MAR]`。
3. **箭头方向**：数据源在左，目标在右（如 `A -> B`）。

---

#### 🌟 第一部分：统写阶段（所有指令闭眼照抄，直接拿基础分）

无论题目让你写哪一条指令，前两个周期（取指和译码）绝对是一模一样的！考试时直接默写：

*   **1. 取指周期 (IF - Instruction Fetch)**
    *   `PC → MAR`  *(把PC的值发给地址寄存器)*
    *   `M(MAR) → MDR`  *(去主存取出指令代码放入数据寄存器)*
    *   `(PC) + 4 → PC`  *(PC自动加4，指向下一条)*
    *   `MDR → IR`  *(把指令送入指令寄存器，准备解码)*

*   **2. 译码/读寄存器周期 (ID - Instruction Decode)**
    *   `OP(IR) → CU`  *(操作码送入控制单元)*
    *   `R[rs] → A` , `R[rt] → B`  *(将源寄存器的值读出，放到暂存器A和B中)*
    *   `SignExt(imm) → Imm`  *(将指令中的16位立即数进行符号扩展，备用)*

---

#### 🌟 第二部分：分支阶段（根据指令类型写 EX, MEM, WB）

这才是拉开分差的地方！仔细看这 4 种核心指令的标准书写方式，**空缺的阶段直接写“无”**。

**【类型 1：R型算术指令 (以 `add rd, rs, rt` 为例)】**
*   **3. 执行周期 (EX)**
    *   `(A) + (B) → ALUout`  *(ALU计算两数之和，放入输出暂存器)*
*   **4. 访存周期 (MEM)**
    *   **无**  *(算术指令不碰内存)*
*   **5. 写回周期 (WB)**
    *   `(ALUout) → R[rd]`  *(把ALU算出的结果写回目的寄存器rd)*

**【类型 2：取数指令 (以 `lw rt, imm(rs)` 为例) - 步骤最全的指令】**
*   **3. 执行周期 (EX)**
    *   `(A) + (Imm) → ALUout`  *(基址加偏移量，计算出真实的物理地址)*
*   **4. 访存周期 (MEM)**
    *   `(ALUout) → MAR`  *(算好的地址给MAR)*
    *   `M(MAR) → MDR`  *(从主存中把数据读出来)*
*   **5. 写回周期 (WB)**
    *   `(MDR) → R[rt]`  *(把主存读出的数据，写回目的寄存器rt)*

**【类型 3：存数指令 (以 `sw rt, imm(rs)` 为例)】**
*   **3. 执行周期 (EX)**
    *   `(A) + (Imm) → ALUout`  *(算物理地址，和lw一样)*
*   **4. 访存周期 (MEM)**
    *   `(ALUout) → MAR`  *(目标地址给MAR)*
    *   `(B) → MDR`  *(把之前读出来的rt的值，放入MDR)*
    *   `(MDR) → M(MAR)`  *(把MDR的数据写入主存)*
*   **5. 写回周期 (WB)**
    *   **无**  *(不需要往寄存器写东西)*

**【类型 4：条件跳转指令 (以 `beq rs, rt, label` 为例)】**
*   **3. 执行周期 (EX)**
    *   `(PC) + (Imm << 2) → Target`  *(计算跳转目标地址：PC加上左移两位的立即数)*
    *   `(A) - (B) → Zero`  *(ALU将两寄存器相减，产生零标志位)*
*   **4. 访存周期 (MEM)**
    *   **无**
*   **5. 写回周期 (WB)**
    *   `If (Zero == 1) Target → PC`  *(如果相减结果为0，就把目标地址写给PC，完成跳转)*

---

### 💡 为什么这么写绝对能拿满分？

1. **逻辑严密**：它完美体现了 MIPS 经典的“五级流水/五个状态”概念，每一阶段干什么极其清晰。
2. **符号标准**：使用了国内高校通用的 `R[x]`（代表寄存器）和 `M(x)`（代表主存），规避了容易产生歧义的括号连用。
3. **化繁为简**：把你之前觉得“复杂”的步骤，变成了“填空题”。IF 和 ID 直接默写，后三步根据指令的“语义”填入。
