### 考点一：SRAM 与 DRAM 到底是个啥？（必考辨析）

老师在选择题里最喜欢混淆这两兄弟的物理特性。

* **SRAM（静态随机存储器）：** 靠**触发器（锁存器）**存数据。有电数据就在，速度**极快**，但电路复杂、造价高。**（通途：用来做 Cache）**
* **DRAM（动态随机存储器）：** 靠**电容**存数据。电容会漏电，所以必须**定期刷新（维持数据）**，速度慢，但集成度高、便宜。**（用途：用来做主存/内存条）**
* **【秒杀黑话】**：只要看到“需要刷新”、“地址线复用（引脚减半）”，闭眼选 **DRAM**；看到“作Cache”、“不用刷新”，闭眼选 **SRAM**。

#### 🎯 必考真题直接看：

**【单选】** 某计算机的主存采用 DRAM 芯片，而高速缓冲存储器（Cache）采用 SRAM 芯片。下列关于这两者的叙述中，错误的是（ **B** ）。
A. SRAM 靠触发器存储信息，只要不断电，信息就不会丢失
B. DRAM 靠电容存储信息，由于电容会漏电，因而闪存（Flash）也需要定期刷新
C. DRAM 比 SRAM 的集成度高、功耗低、成本低
D. DRAM 采用了地址线复用技术，其引脚数约等于地址位数的一半

> **大白话挖坑解析：** 选 B。注意看，B 选项的前半句是对的，但后半句移花接木，变成了“闪存（Flash）也需要刷新”。闪存（比如你的 U 盘、固态硬盘）属于非易失性 ROM，断电都不丢数据，压根不需要刷新！只有 DRAM 才需要刷新。

### 考点二：DRAM 的“刷新”到底怎么刷？（高频判断题坑点）

* **刷新的单位：** 必须是“按行刷新”！绝对不是按字节或者按字刷新。
* **刷新对程序员透明：** 刷新是由内存里的硬件电路自己默默完成的，高级语言程序员、甚至操作系统都**不需要管它（对其透明）**。
* **刷新方式：** 1.  *集中刷新*：集中在一个时间把所有行刷完，这段时间内存罢工，产生“死时间（死区）”。
2.  *分散刷新*：把刷新分散到每个工作周期，没有死时间，但整体变慢了。

#### 🎯 必考真题直接看：

**【判断】** DRAM 芯片为了防止信息丢失必须进行定期刷新，刷新操作通常是以存储字或字节为单位进行的，且必须由 CPU 发出刷新指令才能完成。（ **错误 ×** ）

> **大白话挖坑解析：** 一句话里挖了两个巨坑。第一，刷新是以 **“行”** 为单位；第二，刷新是 **硬件自动完成** 的，不需要 CPU 发指令，CPU 甚至都不知道它在刷新。

### 考点三：Cache 的基本原理与替换算法（必考概念）

* **为什么需要 Cache？** 因为 CPU 太快了，主存（DRAM）太慢了，CPU 每次等内存送数据都会急死。为了解决“速度不匹配”，就在中间放一个快如闪电的 Cache（SRAM）。
* **Cache 为什么能有用？** 因为程序存在“局部性原理”：
* *时间局部性*：现在用到的变量，待会儿马上还要用（比如循环体、计数器）。
* *空间局部性*：现在用到的内存单元，它隔壁相邻的单元马上也要被用到（比如数组、顺序执行的代码）。


* **替换算法（LRU）：** 当 Cache 满了，新数据要进来，该扔掉谁？国内考试最爱考 **LRU（近期最少使用）算法**。它把“最近最长一段时间没被访问过”的旧块扔掉。它的理论依据就是**时间局部性**。

#### 🎯 必考真题直接看：

**【单选】** Cache 能够大幅度提高计算机系统的整体访问速度，其理论支撑是程序访问的局部性原理。其中，旨在将刚刚访问过的数据在不久的将来再次访问的是（ **A** ）。
A. 时间局部性
B. 空间局部性
C. 读写局部性
D. 数据局部性

> **大白话挖坑解析：** 选 A。抓住关键词“刚刚访问过”、“不久的将来再次访问”，这强调的是**时间**轴上的重复，所以是时间局部性。如果是“访问周围紧邻的数据”，那才选空间局部性。

### 考点四：Cache 与主存的映射方式（必考计算与概念）

因为 Cache 很小，主存很大，主存的数据放到 Cache 哪个位置，需要规矩。大题如果考过计算，选择题就会考以下概念辨析：

1. **直接映射（死板的一对多）：** 主存的某个块只能去 Cache 的固定位置（主存块号 % Cache总块数）。
* *特点*：硬件最简单，但极易发生位置冲突，导致 Cache 频繁擦写（称为**抖动**）。


2. **全相联映射（极度自由的一对多）：** 主存的块可以去 Cache 的**任意**空闲位置。
* *特点*：位置最灵活，冲突率最低；但找数据时必须全表同时用**硬件比较器**查找，成本最高。


3. **组相联映射（折中方案）：** 把 Cache 分组。组间用直接映射（固定去哪个组），组内用全相联（组里的位置随便挑）。

#### 🎯 必考真题直接看：

**【单选】** 在主存-Cache 层次结构中，若希望块冲突率最低、空间利用率达到最高，应选择（ **B** ）；若希望硬件结构最简单、不需要复杂的相联查找硬件，应选择（ **A** ）。
A. 直接映射方式
B. 全相联映射方式
C. 组相联映射方式
D. 线性映射方式

> **大白话挖坑解析：** 填 B、A。全相联最自由，所以空间利用率最高，冲突最低；直接映射最死板固定，所以硬件做起来最简单。

### 考点五：Cache 控制位（ Valid Bit 有效位）

每个 Cache 块里除了存数据，还必须有辅助标记：

* **有效位（Valid Bit）：** 1 位。刚开机时，Cache 里全是随机乱码，CPU 怎么知道里面有没有装真数据？就看有效位。`1` 代表里面是有效数据，`0` 代表是空块/无效。**这个 1 位的有效位在考试算 Cache 总容量时经常当作必考的隐蔽坑。**

#### 🎯 必考真题直接看：

**【判断】** 计算机刚开机上电或切换进程时，操作系统通常会将 Cache 中所有字块的“有效位（Valid Bit）”全部置为 0。（ **正确 √** ）

> **大白话挖坑解析：** 完全正确！刚开机或者换了进程，老进程的数据就不能用了，硬件或系统会直接把所有有效位清零（置 0），防止 CPU 误把里面的乱码当成有效数据读出来。
