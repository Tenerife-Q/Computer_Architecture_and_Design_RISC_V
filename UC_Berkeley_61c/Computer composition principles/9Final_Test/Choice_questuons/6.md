### 考点一：程序查询 vs 程序中断（高频对比，必考1题）

这是全章最核心的对比，老师最喜欢考 CPU 在这两种方式下的**工作效率和忙闲状态**：

* **程序查询方式（极其低效，CPU 像个苦力）**：
* *控制逻辑*：CPU 只要发一个 I/O 命令，就必须**原地死等（轮询）** I/O 设备准备好。
* *缺点*：CPU 与 I/O 设备**串行工作**。因为外设极慢，CPU 绝大部分时间都在执行“检查外设状态”的废话循环，CPU 效率极低。


* **程序中断方式（高效，CPU 变身甩手掌柜）**：
* *控制逻辑*：CPU 发出 I/O 命令后，**转头去干别的正事**（比如执行其他程序）。等外设数据准备好了，外设主动向 CPU 发送一个**中断请求信号**，CPU 再停下手头的工作去处理外设数据。
* *优点*：CPU 与 I/O 设备**并行工作**，大大节省了 CPU 的宝贵时间。



#### 🎯 必考真题直接看：

**【单选】** 在程序的输入输出控制方式中，（ **A** ）方式会让 CPU 陷入长时间的死等循环，导致 CPU 效率极低；而（ **B** ）方式实现了 CPU 与外设的并行工作。
A. 程序查询方式
B. 程序中断方式
C. 矩阵轮询方式
D. 独立请求方式

> **大白话挖坑解析**：填 A、B。程序查询的本质就是 CPU “每隔一秒问一次外设好了没”，没好就继续等，属于串行死等；程序中断则是“外设好了自己叫我”，属于并行工作。

### 考点二：中断响应过程中的“关中断”与“保护断点”（生死考点）

当外设发来中断请求，CPU 决定响应，硬件会自动完成一连串规定动作。考试最喜欢考：**谁是硬件自动做的？谁是软件（中断服务程序）做的？**

* **硬件自动完成的动作（中断隐指令）：** 1. **关中断**：CPU 立刻关闭中断响应通道（把 $IE$ 标志置 0），防止刚要处理这个中断，又被别的打断。
2. **保护断点**：把当前的 **程序计数器（PC）的值** 强行压入堆栈保存。**（注意：断点就是 PC！不是通用寄存器的值！）**
3. **引出中断服务程序**：把中断服务程序的入口地址传给 PC。
* **软件完成的动作（中断服务程序里写的代码）：**
1. **保护现场**：用 `PUSH` 指令把通用寄存器、状态标志寄存器（PSW）的内容存入堆栈。
2. **开中断**：如果要允许更高级的中断打断自己（实现多重中断），需要手动用代码打开中断。
3. **中断服务**：真正干活的代码（比如读写外设数据）。
4. **恢复现场和退栈**：用 `POP` 指令把通用寄存器的值还原，最后一条指令通常是中断返回指令（如 `IRET`），把断点（老 PC）从堆栈弹回 PC。



#### 🎯 必考真题直接看：

**【单选】** 在程序中断方式中，将当前执行程序的断点（PC内容）保存到堆栈中、以及将中断服务程序的入口地址送入PC的动作，分别是由（ **C** ）实现的。
A. 软件、硬件自动
B. 软件、软件
C. 硬件自动、硬件自动
D. 硬件自动、软件

> **大白话挖坑解析**：选 C。请死死记住：**保存 PC（断点）是硬件自动做的（叫中断隐指令）**。因为如果靠软件代码去存，代码本身就会改变 PC 的值，那就套牢了。而保存通用寄存器（保护现场）才是中断服务程序这个软件进场后干的第一件事。

### 考点三：单重中断 vs 多重中断（嵌套中断）

* **单重中断**：CPU 在处理一个中断时，**打死也不允许别的人打断它**。它的执行流程里，保护现场期间是**关中断**状态，直到处理完、恢复现场准备回家了才开中断。
* **多重中断（嵌套中断）**：高级别的中断**可以打断**正在执行的低级别中断。
* **多重中断能成功的两个必备硬件/软件条件：**
1. **触发器允许**：中断源的优先级要够高。
2. **代码配合**：中断服务程序在保护完现场后，**必须提前用代码执行“开中断”指令**。如果没有这句开中断代码，再高级的中断也进不来，就退化成单重中断了。



#### 🎯 必考真题直接看：

**【判断】** 只要系统中的外设划分了不同的中断优先级，当 CPU 正在处理低优先级中断时，高优先级的中断请求就一定能够强行打断 CPU 从而实现中断嵌套。（ **错误 ×** ）

> **大白话挖坑解析**：经典大坑！就算外设优先级高，如果低优先级的中断服务程序进入之后**没有手动用代码“开中断”**，CPU 的中断通道依然是关闭的，高优先级外设只能在外面干等，无法实现嵌套。

### 考点四：多重中断的“屏蔽字”（唯一必考计算小考点）

多重中断中，每个中断源都有一个“屏蔽字”，用来决定“我正在干活时，能屏蔽掉谁的请求”。

* **规则**：
1. 屏蔽字里对应位是 `1` 意味着**屏蔽（不允许它打断我）**。
2. **任何中断源对自己对应的位必然是 1**（自己肯定不能打断自己）。
3. **优先级高**的中断源，其屏蔽字里的 `1` 越多。因为高优先级的可以屏蔽掉所有比它低的人。



#### 🎯 必考真题直接看：

**【单选】** 设某机有 4 个中断源 A、B、C、D，其硬件排队判优优先级次序为 A > B > C > D。现要求将多重中断的处理优先级修改为 D > A > C > B。则中断源 C 对应的屏蔽字（从高位到低位依次对应 A、B、C、D）应当为（ **D** ）。
A. 1111
B. 0110
C. 1010
D. 0110

> **大白话挖坑解析**：选 D。根据题目要求的实际处理优先级 D > A > C > B：
> * C 的优先级比 B 高，所以 C 可以屏蔽 B $\to$ B位填 1。
> * C 必须能屏蔽自己 $\to$ C位填 1。
> * C 的优先级比 D 和 A 低，所以 C 无法屏蔽 D 和 A $\to$ A位和D位填 0。
> * 串起来就是：A=0, B=1, C=1, D=0，即 `0110`。
