这部分在考试中极少考纯默写，而是考**计算“有效地址（EA, Effective Address）”**。
*   **什么是 EA？** 指令里写的地址往往是个“假地址”或“半成品”，CPU 必须经过一套加法或查表，算出数据真正存放在内存中的**最终物理地址**，这个最终地址就叫 EA。

我们直接用一个**“满分密码本”**，把 7 种寻址方式的 EA 计算公式列出来。你只要会查表，这 8 分就是白给的。

---

### 🗝️ 寻址方式“密码本”（求 EA 公式）

假设指令中给出的形式地址字段为 **`A`**。
*(括号 `( )` 在计组里表示“去这里面取值”)*

1.  **直接寻址 (Direct)**
    *   **公式：$EA = A$**
    *   *大白话：* A 就是真正的内存地址，直接去内存地址 A 拿数据。
2.  **一次间接寻址 (Indirect)**
    *   **公式：$EA = (A)$**
    *   *大白话：* A 只是个指针。你去内存地址 A 拿出来的东西，依然是个地址，你要拿这个新地址再去内存查一次。（访存 2 次）
3.  **寄存器间接寻址 (Register Indirect)**
    *   **公式：$EA = (R)$**
    *   *大白话：* A 指定了一个寄存器 R。寄存器 R 里面装的就是真实的内存地址 EA。（这就是 C 语言里的指针寻址）。
4.  **基址寻址 (Base)** —— （国内极爱考概念选择题！）
    *   **公式：$EA = (BR) + A$**
    *   *大白话：* BR 是基址寄存器。**基址寄存器由操作系统统一管理**（用于给程序分配在内存中的大片起始空地），程序员不能动它。形式地址 A 作为偏移量。
5.  **变址寻址 (Indexed)** —— （国内极爱考概念选择题！）
    *   **公式：$EA = (IX) + A$**
    *   *大白话：* IX 是变址寄存器。**变址寄存器由程序员自己控制**（就像 `for(int i=0)` 里面的 `i`）。A 作为数组的起始基准地址。专门用来遍历数组！
6.  **相对寻址 (PC-Relative)**
    *   **公式：$EA = (PC) + A$**
    *   *大白话：* 以当前 PC（程序计数器）为基准，往前或往后跳 A 个距离。专门用于写 `if/else` 或 `while` 的分支跳转指令（也就是你熟悉的 `beq`）。

*(注意：立即寻址 Immediate 和 寄存器寻址 Register 因为操作数就在指令或寄存器里，不需要访问内存，所以没有 EA。)*

### ⚔️ 实战测试：8 分标准期末真题

国内考试考这道题，一定会给你一个表格或者一段条件，让你分别求不同寻址方式下的 EA 或者最终取出的操作数。

**【考场原题模拟】**
假设某计算机字长 16 位，当前 CPU 中的 **`PC = 200H`** （这里的 PC 已经指向了下一条要执行的指令），变址寄存器 **`IX = 100H`**，基址寄存器 **`BR = 300H`**。
正在执行的指令中，形式地址字段 **`A = 50H`**。

部分内存中的数据如下表所示：
| 内存地址 | 存放的数据 |
| :--- | :--- |
| `50H` | `0010H` |
| `150H` | `1500H` |
| `250H` | `2500H` |
| `350H` | `3500H` |

**【你的任务】**
请根据上面的条件，计算以下 4 种寻址方式的 **有效地址 EA** 和 **最终取出的真实操作数（数据）**：

1.  **直接寻址**：
    *   EA = ?
    *   取出的数据 = ?
2.  **一次间接寻址**：
    *   EA = ?
    *   *(提示：先去找 EA，再根据 EA 看看该去内存哪个格子里拿数据)*
3.  **变址寻址**：
    *   EA = ?
    *   取出的数据 = ?
4.  **相对寻址**：
    *   EA = ?
    *   取出的数据 = ?

*(全部用十六进制计算，直接用加法拼接即可。把你的推断发给我！)*

### ✍️ 终极对账单：4 大寻址方式逐个破译

#### 1. 直接寻址 (Direct)

* **公式应用**：$EA = A$
* **计算过程**：形式地址 $A = 50H$，所以有效地址直接就是 $50H$。
* **按图索骥**：去内存看 `50H` 格子里装的数据，表中写着 `0010H`。
* **🎯 最终答案**：
* **EA = 50H**
* **取出的数据 = 0010H**



---

#### 2. 一次间接寻址 (Indirect)

* **公式应用**：$EA = (A)$
* **计算过程**：先去内存地址 $A = 50H$ 的格子里看一眼。`50H` 里面装的是 `0010H`。根据间寻定义，这个拿出来的 `0010H` 才是真正的有效地址。所以 $EA = 0010H$。
* **按图索骥**：接下来应该去内存地址 `0010H` 去拿最终的真实操作数。
* **⚠️ 考场细节提醒**：在给出的部分内存数据表中，**并没有提供地址 `0010H`（或 `10H`）对应的数据**。
* **严格按表作答**：数据 = **`存储单元[0010H]中的内容`**。
* **如果是题目笔误（常见排版情况）**：如果表中 `50H` 对应的 `0010H` 其实是想印刷成 `0150H`、`0250H` 或 `0350H` 之一的话，答案就会顺藤摸瓜变成对应行的值（例如：若 $EA = 150H$，则数据 = `1500H`）。


* **🎯 最终答案**（严格基于原表数据）：
* **EA = 0010H**
* **取出的数据 = [0010H] 单元的内容**



---

#### 3. 变址寻址 (Indexed)

* **公式应用**：$EA = (IX) + A$
* **计算过程**：取出变址寄存器里的值 $IX = 100H$，加上形式地址 $A = 50H$。
十六进制加法：$100H + 50H = 150H$。所以有效地址 $EA = 150H$。
* **按图索骥**：去内存看 `150H` 格子里装的数据，表中写着 `1500H`。
* **🎯 最终答案**：
* **EA = 150H**
* **取出的数据 = 1500H**



---

#### 4. 相对寻址 (PC-Relative)

* **公式应用**：$EA = (PC) + A$
* **计算过程**：由于题目特别标注了“这里的 PC 已经指向了下一条要执行的指令”，所以我们不需要再额外执行 $(PC) + \text{字长}$ 的操作，直接读取当前的 $PC = 200H$。
加上形式地址 $A = 50H$，十六进制加法：$200H + 50H = 250H$。所以有效地址 $EA = 250H$。
* **按图索骥**：去内存看 `250H` 格子里装的数据，表中写着 `2500H`。
* **🎯 最终答案**：
* **EA = 250H**
* **取出的数据 = 2500H**
