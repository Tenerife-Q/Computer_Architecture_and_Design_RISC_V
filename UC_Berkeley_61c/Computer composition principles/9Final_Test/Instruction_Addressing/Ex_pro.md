国内期末考试在“寻址方式”的大题上，最喜欢在两个地方挖深坑：一个是“相对寻址中 PC 到底加不加字长”**，另一个是**“变址/基址/间寻的复合套娃”。

下面为你精选两道极其典型的期末真题变形，带你把这两个期末高频大坑彻底踩平。

---

### 🚨 题型一：PC 进位陷阱题（相对寻址的终极杀招）

**【真题再现】**
假设某计算机主存**按字节编址**。当前正在执行一条**相对寻址**的转移指令，该指令在内存中的首地址为 `2000H`。已知该指令**占 4 个字节**，其中的形式地址字段（补码表示）值为 `001CH`。

**【考点提问】**

1. 取出该指令后，CPU 中的 PC（程序计数器）的值是多少？
2. 该转移指令的有效地址 EA 是多少？

---

#### 💡 避坑讲解与推导

##### 第一问：斩断常规思维，算准 PC 的真实值

* **核心陷阱**：很多同学一看到“首地址是 `2000H`”，直接就把 $PC = 2000H$ 代入公式计算，这就中了出题人的圈套。
* **硬件真相**：CPU 的工作流程是“先取指令，PC 自动加指令字长，然后再执行指令”。
题目问的是“取出该指令后”，这意味着当 CPU 开始算有效地址（执行阶段）时，PC 早就已经偷偷往前挪了。
* **严密计算**：
* 指令首地址 = `2000H`
* 指令占 4 个字节，且主存按字节编址（意味着一个地址对应 1 个字节）。
* 取出指令后，$PC = 2000H + 4 = \mathbf{2004H}$。



##### 第二问：代入公式，注意十六进制加法

* **公式应用**：相对寻址 $EA = (PC) + A$
* **数据代入**：此时最新的 $(PC) = 2004H$，形式地址 $A = 001CH$。
* **十六进制对齐相加**：
```text
  2 0 0 4 H
+ 0 0 1 C H
-----------

```


* 观察最低位：$4 + C$（十进制的 $4 + 12 = 16$）。十六进制满 16 进 1，所以最低位写 `0`，向高位进 `1`。
* 倒数第二位：$0 + 1 + \text{进位 } 1 = 2$。
* 高位保持不变。
* 得到 $EA = \mathbf{2020H}$。



**🎯 考场满分答案**：

1. PC 的值是 **`2004H`**。
2. 有效地址 EA 是 **`2020H`**。

### 🚨 题型二：全能套娃综合题（多变型查表大题）

**【真题再现】**
某机器字长 16 位，主存**按字编址**。当前 **`PC = 0100H`**（已指向当前指令的下一条），变址寄存器 **`IX = 0050H`**，基址寄存器 **`BR = 2000H`**，通用寄存器 **`R0 = 0060H`**。
当前指令的形式地址字段 **`A = 0040H`**。

部分主存及寄存器单元的内容如下表：

| 地址 / 寄存器 | 存放的数据内容 |
| --- | --- |
| `0040H` | `00A0H` |
| `0050H` | `00B0H` |
| `0060H` | `00C0H` |
| `0090H` | `00D0H` |
| `00A0H` | `00E0H` |
| `2040H` | `00F0H` |

**【考点提问】**
请分别求出以下寻址方式的**有效地址 EA** 和 **最终操作数**：

1. 一次间接寻址
2. 寄存器间接寻址（使用 R0）
3. 基址寻址
4. 变址寻址

---

#### 💡 避坑讲解与推导

##### 1. 一次间接寻址 (Indirect)

* **公式**：$EA = (A)$
* **推导**：形式地址 $A = 0040H$。查表看主存 `0040H` 里面装了什么，查到数据是 `00A0H`。所以真正的有效地址 $EA = 00A0H$。
* **找操作数**：拿着用完的 $EA = 00A0H$ 再次去查主存，发现 `00A0H` 单元里装的是 `00E0H`。
* **🎯 答案**：$EA = \mathbf{00A0H}$，操作数 = \mathbf{00E0H}。

##### 2. 寄存器间接寻址 (Register Indirect)

* **公式**：$EA = (R0)$
* **推导**：指令点名了 R0 寄存器。查表看 `R0` 里面装的值是 `0060H`。所以真实地址直接就是寄存器里的值，$EA = 0060H$。
* **找操作数**：去主存看 `0060H` 单元，里面装的数据是 `00C0H`。
* **🎯 答案**：$EA = \mathbf{0060H}$，操作数 = \mathbf{00C0H}。

##### 3. 基址寻址 (Base)

* **公式**：$EA = (BR) + A$
* **推导**：读取基址寄存器 $(BR) = 2000H$，加上形式地址 $A = 0040H$。
$2000H + 0040H = 2040H$。所以 $EA = 2040H$。
* **找操作数**：去主存看 `2040H` 单元，里面装的数据是 `00F0H`。
* **🎯 答案**：$EA = \mathbf{2040H}$，操作数 = \mathbf{00F0H}。

##### 4. 变址寻址 (Indexed)

* **公式**：$EA = (IX) + A$
* **推导**：读取变址寄存器 $(IX) = 0050H$，加上形式地址 $A = 0040H$。
$0050H + 0040H = 0090H$。所以 $EA = 0090H$。
* **找操作数**：去主存看 `0090H` 单元，里面装的数据是 `00D0H`。
* **🎯 答案**：$EA = \mathbf{0090H}$，操作数 = \mathbf{00D0H}。

---

### 🛠️ 考前闭眼复盘流

做完这两道综合题，你会发现万变不离其宗：

1. **只要考相对寻址**：一定要瞪大眼睛看清指令占几个字节、主存是不是按字节编址。如果是，**雷打不动先让 PC 加上指令字节数**再套公式。
2. **只要考其余寻址**：严格在草稿纸上写下公式（如 $BR+A$ 或 $IX+A$），**先算地址，再查表格**。表格里的有些干扰项（比如这道题里的 `0050H` 单元的 `00B0H`）是故意诱惑你直接去加的，千万别看错行。

这两类期末大题的杀招，现在是不是已经完全难不倒你了？
